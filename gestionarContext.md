Documentación de Integración API Gestionar - APIs externas para clientes Este documento resume de forma clara y operativa las funcionalidades de la API de Gestionar Logística, con el objetivo de evaluar e implementar su integración con nuestra web/e-commerce. 1. Objetivo de la integración Integrar nuestra web con Gestionar Logística para: Centralizar ventas provenientes de distintos marketplaces. Visualizar pedidos listos para despachar. Gestionar paquetes logísticos. Crear y administrar pickups (retiros). Consultar estados, reportes y tracking. La integración se realiza vía API REST, autenticada mediante API Key. 2. Autenticación Todos los endpoints requieren el siguiente header: secret-token-key: <API_KEY_DEL_CLIENTE> Sin este header, la API devuelve error de autenticación. 3. Módulo Ventas (Sales) Endpoint base GET /api/external-client/sales/{platform_id} Este endpoint permite consultar ventas según el marketplace de origen. Detectar ventas aún no despachadas. Unificar pedidos de múltiples plataformas. Base para creación de paquetes y pickups. 4. Módulo Paquetes (Packages) Consultar paquetes (Reporte) GET/api/external-client/packages Funcionalidad Permite listar y filtrar paquetes logísticos ya creados, con información de estado, tracking e historial.
